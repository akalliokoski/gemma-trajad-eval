<Person2>[chuckle] Exactly. Phase 0 says the main gap is data quality discipline, not missing infrastructure.</Person2>

<Person1>[gasp] Which means raw-format awareness is not a detail. It is a foundation. If inspection code assumes role and content, it is already drifting away from the real data.</Person1>

<Person2>Exactly. Keep the message schema simple. Just enrich metadata. That follows YAGNI and avoids forcing every downstream component to change.</Person2>
<Person1>[sigh] The really interesting part is perturbation quality. Phase 0 made it pretty obvious that direct perturbation alone can create unrealistic bad traces.</Person1>
<Person2>Yes. Traj A D is strongest when anomalies remain coherent after the first wrong step. This repo is not ready for full perturb-and-complete yet, but it is ready for a lightweight coherence screen.</Person2>
<Person1>So instead of adding an L L M continuation stage immediately, we add a deterministic filter that rejects obviously broken perturbed outputs.</Person1>

<Person1>[chuckle] So the dataset becomes easier to analyze without becoming more complicated to generate.</Person1>
<Person2>Exactly. Small field, big clarity.</Person2>

<Person2>Yes. That is a classic home-lab win: improve realism before increasing rule count.</Person2>
