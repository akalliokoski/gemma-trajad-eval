What makes this repo different?

  • Hermes is the operator, not just a chat assistant.
  • Each implementation task should also create human-readable learning material.
  • The original TrajAD direction remains, but now as one module in a broader build-and-learn workflow.
  • Execution is intentionally split across the VPS control plane, the user's Apple Silicon Mac, and future Modal GPU jobs.

Process in plain English

Step 1 — collect traces

  • Pull Hermes trajectories from Hugging Face.
  • Keep the filtered dataset as the main substrate.

Step 2 — normalize and perturb

  • Convert raw traces into a consistent internal trajectory format.
  • Create synthetic anomalies such as wrong tool choice, bad args, skipped steps, and repeated steps.

Step 3 — build training sets

  • Write train/dev/test JSONL splits.
  • Reformat them into binary, localize, or joint SFT chat examples.

Step 4 — train and evaluate

  • Fine-tune Gemma with LoRA on Apple Silicon first.
  • Measure detection, localization, anomaly type accuracy, and JSON validity.
