# Home AI Lab Principles

## Definition
These principles define how this repository should behave as the first project in the user's home AI lab: use best practices, avoid over-engineering, prefer clever community-loved tools, and keep the system practical and elegant.

## Why it matters in this repo
This page is the operating philosophy for tool and architecture choices across `gemma-trajad-eval`. It keeps the project ambitious enough to teach real ML/agent engineering, while resisting platform sprawl and premature infrastructure.

## Practical implications
- Prefer simple, well-liked tools with strong communities over bespoke frameworks.
- Add complexity only when it immediately improves quality, reproducibility, or developer leverage.
- Treat the repo as a flagship home-lab project: trustworthy, teachable, and maintainable.
- Keep documentation and the wiki current as implementation advances.
- Favor elegant script-first workflows before introducing heavier services or orchestration layers.
