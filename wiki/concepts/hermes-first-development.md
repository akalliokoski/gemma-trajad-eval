# Hermes-First Development

## Definition
Hermes-first development is the operating model for this repository: Hermes performs the implementation work directly and generates learning material alongside the work.

## Why it matters in this repo
This concept is the main scope change of the project. It reframes the repository from a single experiment into an applied lab for agentic software engineering, data engineering, fine-tuning, and evaluation.

## Practical implications
- Every meaningful task should combine execution with documentation.
- The wiki should accumulate project knowledge continuously instead of leaving it only in chat logs.
- The original trajectory-anomaly project remains an execution track inside the broader system.
- As the first project in the user's home AI lab, implementation choices should follow best practices without drifting into over-engineering.
