# Hermes Agent

## Overview
Hermes Agent is the primary execution system for this repository. It is expected to perform real work with tools and skills rather than merely suggest what to do next.

## Why it matters here
The project pivot makes Hermes the default operator for software development, data engineering, fine-tuning, evaluation, and documentation. This turns the repo into both an implementation workspace and a study environment.

## Practical implications
- Substantial tasks should start with skill loading and repo inspection.
- Hermes should leave behind docs, plans, and study artifacts as it works.
- Reusable project workflows should become skills when they stabilize.
